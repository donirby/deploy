@font-face syntax

How @font-face works

By using the @font-face rule we can enable the use of web fonts in our sites. Since icon fonts are just another web font, it's important that you understand how web fonts work before deploying icon fonts in your projects.

Although the syntax has evolved over the years, here’s the currently accepted standard for @font-face syntax. Please be aware that there are multiple variations on this syntax.

@font-face {
    font-family: 'font name';
 	src: url('font.eot');
    src: url('font.eot?#iefix') format('embedded-opentype'),
         url('font.woff') format('woff'),
         url('font.ttf') format('truetype'),
         url('font.svg#chunky_mobile') format('svg');
    font-weight: normal;
    font-style: normal;
}

‘Font name’ would be the name you use when defining the font-family property, and the multiple src attributes (and values) deal with the tricky nature of browser support for multiple font formats. Font-weight and font-style are used to define the base weight and style of the font.
