Class-based solutions

Creating efficient icon font class styles

Creating class-based icon font deployment can be tedious. Each icon will need an individual class assignment in order to display properly. This can also result in needless repetition of code through visual styling appearing in each declaration. The upside is that you can use class names that are meaningful rather than the encoding values, which makes it easier to remember which class to apply for each icon. You can make this process more efficient through the use of attribute selectors. These selectors allow you to write class definitions that chain together with prefixes and suffixes to apply icons. Parent classes can assign font-family declarations and normalize styling, while individual suffixes can assign specific icons. You can take this even further by adding specific visual styling suffixes as well.

Example


*[class*="icon"]:before {
	font-family: 'icon font';
    */ normalize */
    }
*[class*="-search"]:before {
	content: '\e001';
    }
*[class*="-big"]:before {
	font-size: 2em;
    }
    -------------------------------
<span class="icon-search"></span>
<span class="icon-search-big"></span>
