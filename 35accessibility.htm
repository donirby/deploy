Improving accessibility

Making icons more accessible

Using icon fonts can cause problems with accessibility. In some cases the injection of additional content on the page can cause screen readers to read icon content that make no sense to the user. In other cases you might display an icon by itself with no accessible text, essentially hiding it from some users. To control the accessibility of icons we can use the aria-hidden attribute to hide generated content from screen readers.  Because this property hides all content of an element we can't apply the data–icon attribute to parent elements like paragraphs. Instead we have to add a little additional markup to our HTML wherever we want an icon to appear. The CSS property speak can also be used to hide content although it's not well supported yet. To ensure a standalone icon has descriptive text we can add text to the content of the icons element and then hide it visually. This text will be read by screen readers but not visible in the layout itself.

Example


span[data-icon]:before {
	font-family: 'icon font';
	content: attr(data-icon);
	speak: none;
    */ normalize */
    }
    -------------------------------
<span data-icon="&#xE001;" aria-hidden="true"></span>
